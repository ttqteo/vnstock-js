# v1.3.3 Plan — VCI GraphQL → REST Migration

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (or superpowers:subagent-driven-development if running in same session) to implement this plan task-by-task. Steps use checkbox syntax for tracking.

**Goal:** Sửa lỗi VCI broken trong vnstock-js bằng cách port migration GraphQL → REST mà `vnstock` Python v3.5.2 (commits `d87da7be` + `3bf3519b`) đã làm. Áp dụng cho 3 module: Listing (industries/ICB), Company (overview), Financial (balance sheet, income statement, cash flow, ratio).

**Architecture:** Giữ adapter pattern (`VciAdapter`). Bỏ tất cả GraphQL calls; thay bằng REST endpoints `trading.vietcap.com.vn/api/...` và `iq.vietcap.com.vn/api/iq-insight-service/...`. Thêm handshake `GET /priceboard` cho session cookies; thêm headers `Device-Id` + cookie `device_id`. Transform configs viết lại theo response shape REST mới.

**Tech Stack:** TypeScript ES5 target, axios qua `fetchWithRetry`, Jest. Không thêm dep mới.

**Scope khẳng định:**
- Patch bump 1.3.2 → 1.3.3 (bug fix, không breaking)
- KHÔNG động vào CLI, MCP, easy-mode, watchlist
- KHÔNG refactor adapter interface
- KHÔNG đụng Realtime, Calendar, Directory, Commodity

## New endpoint map

| Module / Method | Old (GraphQL) | New (REST) | HTTP |
|---|---|---|---|
| Listing: industries info | `query CompaniesListingInfo {…}` | `https://iq.vietcap.com.vn/api/iq-insight-service/v2/company/search-bar?language=1` (vi) / `=2` (en) | GET |
| Listing: ICB codes | `query ListIcbCode {…}` | `https://iq.vietcap.com.vn/api/iq-insight-service/v1/sectors/icb-codes` | GET |
| Company: overview (merged) | `query Query($ticker,$lang)` (8 sub-fields) | Split sang nhiều REST: `/details`, `/{symbol}/shareholder-structure`, `/{symbol}/shareholder`, `/v1/events?ticker=`, `/v1/news?ticker=` | GET |
| Financial: balance sheet | `query CompanyFinancialRatio …` | `https://iq.vietcap.com.vn/api/iq-insight-service/v1/company/{symbol}/financial-statement?section=BALANCE_SHEET` | GET |
| Financial: income statement | same query | same URL, `section=INCOME_STATEMENT` | GET |
| Financial: cash flow | same query | same URL, `section=CASH_FLOW_STATEMENT` | GET |
| Financial: ratio | same query | `https://iq.vietcap.com.vn/api/iq-insight-service/v1/company/{symbol}/statistics-financial` | GET |
| Financial: metrics mapping | `query ListFinancialRatio` | `https://iq.vietcap.com.vn/api/iq-insight-service/v1/company/{symbol}/financial-statement/metrics` | GET |

**Handshake (bắt buộc trước Financial calls):** `GET https://trading.vietcap.com.vn/priceboard` (timeout 10s) — trả về cookies `device_id` cần cho subsequent requests.

## Risks / Open questions

- **Rate limit khi handshake**: nếu mỗi adapter instance tạo handshake riêng và user khởi tạo nhiều adapter, có thể bị 429. Mitigation: handshake state global (module-level), không per-instance.
- **Cookie expiration**: nếu `device_id` cookie có TTL ngắn, cần retry handshake khi 401/403. Mitigation: catch error → reset `handshaked = false` → retry 1 lần.
- **Response shape variance**: Python source pinned ở commit cụ thể; nếu Vietcap đổi shape sau đó, fixtures sẽ stale. Mitigation: viết tests integration nhẹ kiểm key fields exist, không assert full shape.
- **CORS browser**: REST endpoints có thể không CORS-friendly. Out of scope cho v1.3.3 — vnstock-js chính chạy Node; browser users tự lo proxy.
