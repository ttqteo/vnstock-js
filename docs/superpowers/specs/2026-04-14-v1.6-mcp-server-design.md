# v1.6.0 — MCP Server Design Spec

**Date:** 2026-04-14
**Status:** Draft, pending user review
**Target release:** v1.6.0 (sau khi v1.4 + v1.5 ship)

## 1. Overview

Expose `vnstock-js` làm **MCP (Model Context Protocol) server** để Claude Desktop / Cursor / VS Code / các MCP client có thể query stock data Việt Nam qua tool-calling.

USP: Z (VN-native context) — tool descriptions + data VCI real-time. Claude/LLM khác lo UI + natural language reasoning. Mình chỉ cung cấp thin tool layer trên SDK đã có.

## 2. Scope

### In
- Subcommand mới `vnstock mcp` — stdio MCP server
- 8 core tools (xem Section 5)
- Install guide trong docs cho Claude Desktop / Cursor / VS Code
- Demo video + blog post launch
- Đăng ký trong [MCP registry](https://github.com/modelcontextprotocol/servers)

### Out
- Realtime WebSocket qua MCP — MCP không support streaming natively
- Authentication — VCI public API, không cần
- Screener/fundamentals tools — đẩy sang v1.7 khi SDK có module tương ứng
- Separate npm package `@vnstock-js/mcp` — đơn giản nhất là subcommand (1 install = CLI + MCP)

## 3. Architecture

```
Claude Desktop / Cursor / VS Code
  │  spawn subprocess + stdio
  │  JSON-RPC 2.0
  ▼
vnstock mcp (Node subprocess)
  │  @modelcontextprotocol/sdk
  ▼
Tool handlers (src/cli/mcp/index.ts)
  │  reuse existing SDK 100%
  ▼
vnstock.stock.* / Directory.* / calendar.*
  │
  ▼  
VCI API (+ disk cache + retry)
```

### Nguyên tắc
- **Thin layer** — không logic riêng, chỉ wrap SDK
- **Lazy init** — chỉ gọi `await init()` khi tool cần (search/list/calendar). `get_quote`/`get_history` không cần → startup <500ms
- **Session cache** — MCP sống suốt session Claude, cache in-memory giảm pressure VCI
- **Bilingual tool descriptions** — tiếng Việt + Anh, có examples câu user

## 4. File structure

```
src/cli/mcp/
  index.ts           # MCP server entry, register tools + transport
  tools.ts           # Tool schemas array
  handlers.ts        # Tool execution handlers
  cache.ts           # Session cache for quotes
__tests__/cli/mcp/
  handlers.test.ts   # Unit tests mỗi handler
  schemas.test.ts    # Validate tool schemas
```

Wire vào `src/cli/index.ts`:
```typescript
program
  .command("mcp")
  .description("Start MCP server on stdio (for Claude Desktop / Cursor / VS Code)")
  .action(async () => {
    const { start } = await import("./mcp/index");
    await start();
  });
```

Dynamic import để không tăng startup time cho commands khác.

## 5. Tool schemas (8 core)

### Tool 1: `get_quote`

```typescript
{
  name: "get_quote",
  description:
    "Lấy giá cổ phiếu Việt Nam hiện tại (snapshot). Trả về giá khớp, % thay đổi, khối lượng, giá trần/sàn/tham chiếu. " +
    "Dùng khi user hỏi 'giá VCB hôm nay', 'FPT bao nhiêu', 'cổ phiếu X tăng hay giảm'. " +
    "EN: Get current Vietnam stock quote. Returns matched price, change%, volume, ceiling/floor/reference prices.",
  inputSchema: {
    type: "object",
    properties: {
      symbol: {
        type: "string",
```

## 6. Response format conventions

Mỗi handler return object MCP chuẩn:

```typescript
{
  content: [
    { type: "text", text: "<readable summary tiếng Việt>" },
    { type: "text", text: JSON.stringify(structuredData) }
  ],
  isError?: boolean,
}
```

- **Text readable** cho Claude hiểu nhanh + relay user
- **JSON structured** cho Claude reasoning (so sánh, tính toán)
- **Error case**: `isError: true` + message tiếng Việt gợi ý action ("thử search_symbols để tìm mã đúng")

## 7. Optimizations

### (a) Lazy init

```typescript
let initialized = false;
async function ensureData() {
  if (!initialized) {
    await init();
    initialized = true;
  }
}
```

Gọi `ensureData()` ở đầu handler cần symbols/holidays. `get_quote`/`get_history` skip.

### (b) Session cache quote

```typescript
const quoteCache = new Map<string, { data: PriceBoardItem, t: number }>();
const QUOTE_TTL = 30_000; // 30s

async function getQuoteCached(symbol: string) {
  const hit = quoteCache.get(symbol);
  if (hit && Date.now() - hit.t < QUOTE_TTL) return hit.data;
  const [data] = await vnstock.stock.trading.priceBoard([symbol]);
  if (data) quoteCache.set(symbol, { data, t: Date.now() });
  return data;
}
```

Claude có thể hỏi cùng symbol nhiều lần — cache 30s giảm VCI pressure.

### (c) Parallel bulk calls

`top_movers`:
```typescript
const [gainers, losers] = await Promise.all([
  vnstock.stock.trading.topGainers(),
  vnstock.stock.trading.topLosers(),
]);
```

### (d) Error quality

```typescript
if (!data) {
  return {
    content: [{
      type: "text",
      text: `Không tìm thấy mã "${symbol}". Hãy thử tool search_symbols với query là tên công ty.`
    }],
    isError: true,
  };
}
```

Claude đọc message và tự retry `search_symbols`.

### (e) Startup time

Dynamic import `@modelcontextprotocol/sdk` và `vnstock` SDK chỉ khi `mcp` command được gọi. Các CLI commands khác không tốn time.

## 8. Testing

### Unit tests `__tests__/cli/mcp/handlers.test.ts`
- Mock SDK methods (`vnstock.stock.trading.priceBoard`, etc.)
- Test mỗi handler với valid input → expected content shape
- Test error paths (symbol not found, invalid date, SDK throws)

### Schema tests `__tests__/cli/mcp/schemas.test.ts`
- Each tool schema passes JSON Schema draft-07 validation
- Required fields marked correctly
- Description length > 50 chars (LLM needs context)

### Manual test — MCP Inspector
```bash
npx @modelcontextprotocol/inspector npx vnstock mcp
```
- Click mỗi tool, nhập input thật (VCB, FPT, ...)
- Verify output readable + JSON correct

### End-to-end — Claude Desktop
1. Config `claude_desktop_config.json`
2. Reload Claude Desktop
3. Ask 5 test queries (bilingual):
   - "VCB giá hôm nay" → should call `get_quote`
   - "liệt kê 10 mã tăng mạnh nhất" → `top_movers`
   - "FPT 7 ngày qua thế nào" → `get_history` with range=7d
   - "tìm các ngân hàng niêm yết" → `search_symbols`
   - "ngày 30/4/2026 có giao dịch không" → `is_trade_day`

## 12. Risks

| # | Risk | Mitigation |
|---|---|---|
| R1 | MCP protocol breaking change | Pin `@modelcontextprotocol/sdk` version, monitor [spec changelog](https://spec.modelcontextprotocol.io/) |
| R2 | Claude spam tool calls → VCI rate-limit | Session cache (f) + v1.2 `rateLimitWait` auto-wait |
| R3 | Tool descriptions dịch không tự nhiên → Claude pick sai | User test 20+ câu hỏi, iterate descriptions |
| R4 | Startup time > 1s → Claude timeout | Lazy import, measure với `time vnstock mcp < /dev/null` |
| R5 | MCP registry không accept (no auth, quality bar) | Theo checklist MCP contribution guide |
