**Date:** 2026-04-14
**Parent spec:** [2026-04-10-vnstock-js-v1.3.0-design.md](2026-04-10-vnstock-js-v1.3.0-design.md) Section 5

## Context

Phát hiện từ fixture capture thực tế (2026-04-14, giờ giao dịch 13:32 VN, symbols VCB/FPT/MBB, 45s) lưu tại `__tests__/fixtures/realtime/session-2026-04-14T06-32-07-325Z.jsonl`:

| Observation | Impact on design |
|---|---|
| Server trả ack plain string `"Subscribed stockRealtimeBySymbolsAndBoards"` sau sub | Routing cần xử lý message không phải pipe-delimited cũng không phải JSON |
| Server trả `{"result":"complete","type":"unsub",...}` khi nhận unsub message | Server hỗ trợ unsub thật — hiện code không gửi |
| Không có ping/pong event nào trong 45s | SSI không dùng WebSocket-level heartbeat — ping() hiện tại vô nghĩa |
| Quote pipe-delimited flow liên tục (~3-5 msg/s cho 3 symbols) | Dùng incoming rate làm heartbeat (dead-man's switch) là khả thi |
| Không có handshake riêng | Không cần state machine phức tạp |
| Sub 3 symbols trong 1 message OK, ack sau ~9ms | Không cần batch resubscribe |

Spec cha đoán có handshake, có ping/pong, lo batch. Fixture cho thấy **không**. YAGNI cắt scope theo data thực.

## Decisions

| # | Decision | Rationale |
|---|---|---|
| D1 | Dead-man's switch thay ping/pong | SSI không có WS ping. Message flow liên tục trong giờ giao dịch — silent > 60s = dead |
| D2 | `unsubscribe()` gửi unsub thật (fire-and-forget) | Server support, băng thông phí nếu không gửi |
| D3 | Ack plain string bị ignore | Không đem lại signal gì user cần |
| D4 | Không batch resubscribe | SSI accept nhiều symbol/message, chưa có evidence giới hạn |
| D5 | Không state machine | Không có handshake/auth phase, state cờ `intentionalClose` đủ dùng |
| D6 | Routing by message shape: pipe/JSON/plain | Đơn giản, match data thực |
| D7 | Test strategy: parser fixture + routing unit + smoke script | Cân bằng CI-friendly và coverage lifecycle |
| D8 | `deadManTimeout` option mới, bỏ `heartbeatInterval` + `heartbeatTimeout` | Rename/replace, technically breaking nhưng là implementation detail |

## Architecture

```
┌──────────────────────────────────────────────────────┐
│                  RealtimeClient                       │
│                  (EventEmitter)                       │
│                                                       │
│   connect()  →  _createSocket()                       │
│                       ↓                               │
│                  onmessage                            │
│                       ↓                               │
│            _routeMessage(data)                        │
│            ↙           ↓           ↘                  │
│   has "|"        starts with "{"    plain text        │
│        ↓               ↓                ↓             │
│   parseData()    JSON.parse()       log only          │
│        ↓          ↙       ↘         (e.g. Subscribed) │
│   emit("quote")  result: error:                       │
│                  complete  ...                        │
│                  (ignore)  emit("error")              │
│                                                       │
│   Dead-man timer: reset trong _routeMessage,          │
│                   fire sau deadManTimeout →           │
│                   socket.close() → reconnect          │
└──────────────────────────────────────────────────────┘
```

## Changes summary

### Code (`src/realtime/`)

**`src/realtime/types.ts`:**
- Remove: `heartbeatInterval`, `heartbeatTimeout`
- Add: `deadManTimeout?: number` (ms, default 60000)

**`src/realtime/index.ts`:**
- Remove: `_startHeartbeat`, `_resetHeartbeatTimeout`, `_stopHeartbeat`, `heartbeatTimer`, `heartbeatTimeoutTimer`
- Add: `_resetDeadManTimer`, `_clearDeadManTimer`, `deadManTimer` field
- Rewrite `onmessage` to use `_routeMessage`
- Rewrite `unsubscribe()` to send unsub message to server
- Remove all try/catch around `socket.ping()` (no more ping)

### Message routing logic

```typescript
this.socket.onmessage = function (event: any) {
  var data = event.data || event;
  if (typeof data !== "string") return;

  self._resetDeadManTimer();

  if (data.indexOf("|") !== -1) {
    try {
      self.emit("quote", parseData(data));
    } catch (err) {
      self.emit("error", err instanceof Error ? err : new Error(String(err)));
    }
    return;
  }

  if (data.charAt(0) === "{") {
    try {
      var msg = JSON.parse(data);
      if (msg && msg.error) {
        self.emit("error", new Error(String(msg.error)));
      }
      // {result:"complete",...} and other JSON: silent ignore
    } catch (_e) {
      // Malformed JSON: ignore
    }
    return;
  }

  // Plain string (sub ack): ignore
};
```

### Dead-man's switch

```typescript
private _resetDeadManTimer(): void {
  var self = this;
  this._clearDeadManTimer();
  this.deadManTimer = setTimeout(function () {
    if (self.socket) {
      self.socket.close();
    }
  }, this.deadManTimeout);
}
```

Start in `onopen`, reset in `_routeMessage`, clear in `onclose`/`disconnect`.

### Unsubscribe

```typescript
unsubscribe(symbols: string[]): void {
  for (var i = 0; i < symbols.length; i++) {
    this.symbols.delete(symbols[i]);
  }
  if (this.socket && this.socket.readyState === 1 && symbols.length > 0) {
    this.socket.send(JSON.stringify({
      type: "unsub",
      topic: "stockRealtimeBySymbolsAndBoards",
      variables: { symbols: symbols, boardIds: ["MAIN"] },
      component: "priceTableEquities",
    }));
  }
}
```

## Testing

### `__tests__/realtime-v2.test.ts` (rewrite)

**Suite 1: parseData fixture-based**
- Load `session-2026-04-14T06-32-07-325Z.jsonl`
- Filter `dir === "in"` + `raw.indexOf("|") !== -1`
- Assert every quote parses without throw, has valid symbol/price/bid-ask structure
- Negative: `parseData("bad")` throws `ParseError`

**Suite 2: _routeMessage unit**
- Synthetic messages (not fixture), mock WebSocket
- Pipe quote → emit `quote` once with parsed data
- `{"result":"complete","type":"unsub"}` → no event
- `{"error":"bad"}` → emit `error`
- `"Subscribed ..."` → no event
- Malformed pipe data → emit `error` (parser throws)
- Malformed JSON → no event (silent)

**Suite 3: lifecycle**
- `connect()` → onopen fires → `connected` event + sub sent with symbols
- `subscribe(["X"])` after connected → sub message sent
- `unsubscribe(["X"])` → unsub message sent with correct schema, local Set updated
- Dead-man timer: `jest.advanceTimersByTime(deadManTimeout + 1)` without message → `socket.close()` called
- Dead-man reset: incoming message before timeout → no close
- `disconnect()` → intentional close, no reconnect
- Reconnect path: close unexpected → `reconnecting` event → socket recreated

## Migration (CHANGELOG note)

```
### Breaking (minor)
- `RealtimeClientOptions.heartbeatInterval` và `heartbeatTimeout` bị loại
  (không dùng nữa vì SSI không có WS ping).
- Thay bằng `deadManTimeout` (ms, default 60000) — reconnect nếu không nhận
  message trong khoảng này.

### Fixed
- `unsubscribe()` giờ thực sự gửi unsub message cho server.
- Message routing phân biệt quote/JSON/plain text, không còn nuốt ack.
- Loại heartbeat ping() vô nghĩa, browser env không còn bị reset connection
  oan mỗi 40s.
```

## Out of scope (punted)

- **State machine:** không cần — cờ `intentionalClose` đủ
- **Handshake verify:** SSI không có handshake
- **Batch resubscribe:** SSI accept bulk sub, không có evidence cần batch
- **ETag/Retry-After cho ack:** không liên quan realtime
- **Multi-exchange (HNX/UPCOM in realtime):** chỉ HOSE (MAIN board), giữ như cũ

## Open questions (none blocking)

- Q: Nếu sau này user scale lên 100+ symbols → có bị SSI từ chối không? → Chờ user report, không phòng xa
- Q: Server có bao giờ gửi unprompted reconnect signal không? → Chưa thấy trong fixture, nếu có sau sẽ thêm vào routing
