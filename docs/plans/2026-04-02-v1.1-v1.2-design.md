# vnstock-js v1.1 / v1.2 Design Document

> Created: 2026-04-02
> Status: Approved

## Context

vnstock-js v1.0.1 is functional but needs to transition from "usable" to "long-term SDK foundation". Target audience: Vietnamese developers building dashboards/web apps (hobbyist first, startup-ready later). Base user is small, so breaking changes in 1.1 are acceptable.

## Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Multi-source | Adapter interface only, no new sources in 1.x | Keep scope small, prepare foundation for 2.0 |
| FinancialStatement | Strict base fields + generic extension + index signature | Autocomplete for common fields, backward-compatible |
| Realtime | Event emitter v2, replace old API in 1.1 (no deprecation period) | Small user base, cleaner to cut over immediately |
| Market calendar | Static hardcoded JSON, no API fetch | Simple, reliable, sufficient for alerting/backtest |
| Errors | Full taxonomy with 6 custom classes | Enable typed catch, good DX for production use |

## v1.1.0 — Stable Core + Realtime v2

### 1. Error Taxonomy

**Location**: `src/errors/`

```
VnstockError (base)
├── NetworkError          — timeout, connection refused, DNS
│   └── RateLimitError    — HTTP 429
├── ApiError              — 4xx/5xx (non rate-limit)
├── InvalidSymbolError    — bad ticker symbol
├── InvalidParameterError — bad params (date, period, etc.)
└── ParseError            — unparseable response (realtime, HTML gold)
```

Each error has:
- `.code` — string enum: `"NETWORK_ERROR"`, `"RATE_LIMIT"`, `"API_ERROR"`, `"INVALID_SYMBOL"`, `"INVALID_PARAMETER"`, `"PARSE_ERROR"`
- `.message` — human-readable
- `.cause` — original error
- `.statusCode?` — HTTP status if applicable

**Integration points**:
- `pipeline/fetch.ts` — wrap axios errors into `NetworkError` / `RateLimitError` / `ApiError`
- Core modules — validate inputs, throw `InvalidSymbolError` / `InvalidParameterError`
- Realtime `parseData` — throw `ParseError` on malformed data

### 2. FinancialStatement Typing

```typescript
interface FinancialStatementBase {
  symbol: string;
  year: number;
  quarter: number;
  updatedAt: string;
  revenue?: number;
  grossProfit?: number;
  netIncome?: number;
  totalAssets?: number;
  totalEquity?: number;
  totalDebt?: number;
  operatingCashFlow?: number;
}

interface FinancialStatement<T = {}> extends FinancialStatementBase {
  [key: string]: unknown;
}
```

### 3. Adapter Interface

**Location**: `src/adapters/`

```typescript
interface DataAdapter {
  readonly name: string;
  readonly source: string;
  fetchQuoteHistory(params: QuoteHistoryParams): Promise<QuoteHistory[]>;
  fetchPriceBoard(symbols: string[]): Promise<PriceBoardItem[]>;
  fetchListing(params: ListingParams): Promise<ListedSymbol[]>;
  fetchFinancials(params: FinancialParams): Promise<FinancialStatement[]>;
  fetchCompanyProfile(symbol: string): Promise<CompanyProfile>;
}
```

In 1.x only `VciAdapter` implements this. Core modules call through adapter instead of direct URL construction. Public API unchanged.

**Directory**:
```
src/adapters/
  types.ts          — DataAdapter interface
  vci/              — VciAdapter (extracted from current core)
  ssi/              — SSI WebSocket adapter (extracted from realtime)
  btmc/             — Gold/exchange adapters
```

### 4. Realtime v2

**Location**: `src/realtime/` (replaces `src/core/realtime.ts`)

**API**:
```typescript
import { realtime } from 'vnstock-js';

const client = realtime.create({
  symbols: ['VNM', 'FPT'],
  autoReconnect: true,        // default: true
  reconnectInterval: 3000,    // ms, default: 3000
  maxReconnectAttempts: 10,   // default: 10
  heartbeatInterval: 30000,   // ms, default: 30000
});

client.on('quote', (data: RealtimeQuote) => { ... });
client.on('connected', () => { ... });
client.on('disconnected', (reason: string) => { ... });
client.on('reconnecting', (attempt: number) => { ... });
client.on('error', (err: VnstockError) => { ... });

client.connect();
client.subscribe(['HPG', 'MSN']);
client.unsubscribe(['VNM']);
client.disconnect();
```

**Internals**:
- Auto-reconnect with exponential backoff on disconnect
- Heartbeat ping every 30s, 10s pong timeout → reconnect
- Queue subscribe commands if socket not yet open, flush on connected
- Old `VnstockRealtime` API removed entirely (no deprecation — small user base)

## v1.2.0 — Production Ready

### 7. Market Calendar

**Location**: `src/core/market/`, `data/holidays/`

```typescript
import { market } from 'vnstock-js';

market.calendar.isTradeDay('2026-04-02');     // boolean
market.calendar.nextTradeDay('2026-04-02');   // '2026-04-03'
market.calendar.prevTradeDay('2026-04-02');   // '2026-04-01'
market.calendar.holidays(2026);               // string[]

market.calendar.session();
// { exchange: 'HOSE',
//   morning: { open: '09:00', close: '11:30' },
//   afternoon: { open: '13:00', close: '14:45' },
//   ato: '09:15', atc: '14:30' }
```

**Data**: Static JSON files per year in `data/holidays/`. Updated annually via PR.

### 8. Pagination

Standardized pagination for listing + screening endpoints:
- Cursor-based or page-based depending on underlying API
- Consistent `{ data: T[], hasMore: boolean, cursor?: string }` response wrapper

### 9. Rate Limit Handling

- Detect HTTP 429 in `fetchWithRetry`
- Auto-wait using `Retry-After` header if present
- Throw `RateLimitError` if max retries exceeded
- Configurable max wait time
